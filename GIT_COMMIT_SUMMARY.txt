MASSIVE BUG FIX - Competitive quiz submissions now fully working end-to-end!

Competitive quiz submissions appeared successful in UI but never reached Firebase
Firestore, leaving users unable to compete. Zero debug visibility meant diagnosis
was impossible.

ROOT CAUSES FIXED:
1. Duplicate method definitions in competitive-results.page.ts
   (tryAgain, viewLeaderboard, goHome)
   → Prevented dev server compilation, blocked all testing

2. Missing TypeScript bracket notation for router state access
   (property comes from index signature)
   → Fixed with proper state['results'] accessor

3. Firestore injection context warnings in leaderboard.service.ts
   → Wrapped collection() and query() calls inside ngZone.runOutsideAngular()

4. Angular zone management for Firestore operations
   → Added ngZone.run() wrapper for addDoc() operation

5. Zero debug visibility throughout execution chain
   → Added comprehensive logging at 12 strategic points
